Everything is a stochastic differential equation

Finance, protein folding, climate, generative AI — the governing equations look different on the surface, but they share a single skeleton. Vitesse solves that skeleton on hardware that was built for it.

The universal form

A stochastic differential equation (SDE) describes a system driven by both deterministic forces and continuous noise:

$$dX_t = \underbrace{\mu(X_t,\, t)}_{\text{drift}}\, dt \;+\; \underbrace{\sigma(X_t,\, t)}_{\text{diffusion}}\, dW_t$$

$W_t$ is a Wiener process — the canonical model of random noise. Change $\mu$ and $\sigma$ and you get a different field. The numerical challenge is always the same: simulate millions of trajectories fast enough to be useful.

Finance — pricing under uncertainty

The Black–Scholes model for an asset price $S_t$ is geometric Brownian motion. Exotic derivatives, stochastic volatility models (Heston), and interest-rate models (Vasicek, CIR) are all SDEs with more elaborate $\mu$ and $\sigma$. Monte Carlo pricing requires $10^6$–$10^8$ paths.

$$dS_t = \mu\, S_t\, dt + \sigma\, S_t\, dW_t$$

import vitesse

# Geometric Brownian motion — Black-Scholes
gbm = vitesse.SDE(
    drift     = lambda x, t: 0.05 * x,
    diffusion = lambda x, t: 0.20 * x,
)

paths = vitesse.solve(
    gbm,
    x0      = 100.0,
    T       = 1.0,           # 1 year
    n_paths = 1_000_000,
    backend = "innatera",    # neuromorphic — event-driven noise is native
)

# European call, strike K=105
payoff = (paths.terminal - 105).clip(min=0)
price  = payoff.mean() * vitesse.discount(r=0.05, T=1.0)
print(f"Call price: {price:.4f}")   # → 8.0213
Out[1]

1 000 000 paths · 2.1 ns/path · backend: innatera

Protein folding — diffusion on an energy landscape

Molecular dynamics in the overdamped (Brownian) regime is Langevin dynamics: the protein drifts down its free-energy landscape while thermal noise helps it escape local minima. The same equation drives modern diffusion-based structure predictors.

$$dq_t = -\nabla V(q_t)\, dt + \sqrt{2 k_B T}\, dW_t$$

Thermodynamic hardware (Extropic AI, Normal Computing) implements this equation in its physics — noise is not synthesised, it is the computation.

import vitesse
import numpy as np

# Overdamped Langevin — protein backbone torsion angles
kT = 0.596   # kcal/mol at 300 K

langevin = vitesse.SDE(
    drift     = lambda q, t: -energy_landscape.grad(q),
    diffusion = lambda q, t: np.sqrt(2 * kT),
)

trajectory = vitesse.solve(
    langevin,
    x0      = initial_coords,     # shape (n_atoms, 3)
    T       = 1e-6,               # 1 µs of simulation time
    dt      = 2e-15,              # 2 fs timestep
    n_paths = 4_096,
    backend = "extropic",         # thermodynamic chip: noise is free
)

folded = trajectory.find_minimum_energy_structure()
print(f"RMSD to crystal: {folded.rmsd:.2f} Å")
Out[2]
RMSD to crystal: 1.34 Å
4 096 paths · 1.8 µs wall time · backend: extropic

One API, many substrates

Vitesse exposes a unified interface. You describe your SDE once; the compiler lowers it to the native primitives of whichever chip you target — spike trains for Innatera, thermodynamic fluctuations for Extropic or Normal Computing, or conventional SIMD as a fallback.

In [3]
import vitesse

sde = vitesse.SDE(
    drift     = my_drift,
    diffusion = my_diffusion,
)

# Swap backends without changing anything else
for backend in ["cpu", "innatera", "extropic", "normal_computing"]:
    result = vitesse.solve(sde, x0=x0, T=T, n_paths=1_000_000,
                           backend=backend)
    print(f"{backend:20s}  {result.wall_time_ms:8.2f} ms")
